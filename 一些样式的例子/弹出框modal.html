大/小尺寸模态弹出框,可通过给“.modal”增加类名“fade”为模态弹出框增加一个过渡动画效果。

~ data-toggle 默认值modal,必须要在触发元素(如button、a等)上定义的一个属性，而且值只能是modal
~ data-target 用来定义要触发的模态弹出框是哪个。其值可以是“div.modal”元素独有的类名，或者ID名。
1.data-backdrop="false",默认true,是否包含一个背景div（默认灰色）,如果为真，则单击该背景时关闭弹出框，如果取值为static,则单击背景div元素时不会关闭弹出框
2.data-keyboard="false",默认true,按键盘的ESC退出弹出框，
3.data-show 默认true,初始化时，弹出框是否显示
4.href URL路径 ，如果通过a元素来触发模态弹出框，其href 值不是一个已#开头的值，则表示是一个url地址，模态弹出框会先加载其内容，， 然后替换原有的modal-content中的内容。如果href设置了地址，那么 data-target则必须填写制定id

模态弹出框--JavaScript触发时的参数设置(3种)

参数 使用方法描述
toggle$(“#mymodal”).modal(“toggle”) 触发时，反转模态弹出窗的状态。如果模态弹出窗是显示的，则关闭；反之，如果模态弹出窗是关闭的，则显示
show $(“#mymodal”).modal(“show”)触发时，显示模态弹出窗
hide$(“#mymodal”).modal(“hide”)触发时，关闭模态弹出窗

模态弹出框--JavaScript触发时,模态弹出窗还支持四种类型的事件，分别是模态弹出窗的弹出前、弹出后，关闭前、关闭后，具体描述如下：

事件类型调用方法描述
show.bs.modal $('#myModal').on('show.bs.modal', function (e) { // 处理代码... }) 在show方法调用时立即触发（尚未显示之前）；如果单击了一个元素，那么该元素将作为事件的relatedTarget属性
shown.bs.modal $('#myModal').on('shown.bs.modal', function (e) { // 处理代码... })该事件在模态弹出窗完全显示给用户之后（并且等CSS动画完成之后）触发；如果单击了一个元素，那么该元素将作为事件的relatedTarget事件
hide.bs.modal$('#myModal').on('hide.bs.modal', function (e) { // 处理代码... })在hide方法调用时（但还未关闭隐藏）立即触发
hidden.bs.modal$('#myModal').on('hidden.bs.modal', function (e) { // 处理代码... })该事件在模态弹出窗完全隐藏之后（并且CSS动画漂完成之后）触发
